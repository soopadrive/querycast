QueryCast

A user-controlled YouTube subscription feed for Windows. Transparent curation, no algorithm.

What is it?

QueryCast is a Windows desktop app that replaces YouTube's algorithmic home feed with a curation system you fully control. Every video that reaches your feed has passed your filters; every score that ranks them is a transparent linear combination of weighted signals you set.

It is not a YouTube replacement. You still watch videos via YouTube's player. QueryCast replaces discovery — the part where the algorithm decides what to surface.

Status

v0.1 — Stage 1 placeholder. Sign-in and the actual feed land in subsequent stages. The plan is at ~/plans/querycast.html.

BYO credentials model. QueryCast uses your own Google Cloud OAuth credentials so your YouTube API quota is yours alone. First launch shows a setup screen with step-by-step instructions; expect ~15 minutes one-time setup.

Install (end users)

  1. Download QueryCast_0.1.0_x64_en-US.msi from the GitHub release (or the portable .exe from the same release).
  2. Run the installer. Windows SmartScreen will warn: "Microsoft Defender SmartScreen prevented an unrecognized app from starting." Click More info → Run anyway. (The binary is unsigned; an EV cert to skip this is on the v2 wishlist.)
  3. If WebView2 is not already installed (rare on Windows 11, possible on Windows 10), the installer downloads it automatically. Adds ~30 seconds to first install.
  4. Launch QueryCast from the Start menu.
  5. Setup screen appears. Follow the 5-step guide for creating your Google Cloud OAuth credentials. Paste your Client ID and Client Secret. Save.
  6. App restarts into the main view. Sign in with Google when Stage 2 ships.

Build from source (developers)

Prerequisites

  • Rust (stable, ≥1.77): install via rustup.rs
  • Tauri CLI: cargo install tauri-cli --version "^2"
  • Windows builds: Visual Studio Build Tools 2022 with the C++ desktop development workload
  • WebView2 runtime (already on Windows 11; auto-installed via the bundled Evergreen Bootstrapper otherwise)

Run in development

cd querycast
cargo tauri dev

Build a Windows installer

cargo tauri build

Output: src-tauri/target/release/bundle/msi/QueryCast_0.1.0_x64_en-US.msi + portable .exe.

Stages

0Decisions, ADRs, name (QueryCast), sunset criteria — done. 1Tauri scaffold + BYO setup screen + IndexedDB contract — current. 2PKCE auth flow via localhost loopback + refresh tokens. 3Subscriptions list + direct RSS fetch + first feed render. 4Metadata enrichment + tombstones + client quota cap. 5Filter + ranking engine (channel groups, pins, sweet spot). 6UI per locked design + per-video actions + Saved view + profile switcher. 7Settings UI (profiles, channel groups, hidden videos) + Drive backup. 8Soft launch + kill criterion check.

Privacy

All app state — your filters, channel weights, profiles, watched/saved/not-interested videos — lives in IndexedDB on your machine. Backups go to your own Google Drive (in a hidden folder only QueryCast can read). There is no QueryCast server. There is no telemetry. Each user's Google API usage is theirs alone (BYO credentials).
